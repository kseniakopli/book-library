# Разовые скрипты

Утилиты, которые запускаются руками: бэкапы, разовые дозаполнения данных,
генерация артефактов. К работе приложения не относятся — оно их не импортирует.

**Запускать из папки `backend/`** (не из `scripts/`), чтобы относительные пути
и `.env` подхватились правильно:

```
cd backend
python scripts/backup_db.py
```

`_bootstrap.py` кладёт `backend/` в `sys.path` — благодаря ему скрипты могут
делать `import database`, `from models import Book` и т.д.

## Что здесь есть

| Скрипт | Зачем |
|---|---|
| `backup_db.py` | Консистентный снимок `library.db` + ротация (10 последних). Автозапуск — `../run_backup.bat` в Планировщике Windows |
| `dump_openapi.py` | Снимок схемы API → `documentation/openapi.json`. **Обязательно после правок эндпоинтов** — иначе CI-проверка контракта покраснеет |
| `reset_playlist.py` | Сброс ссылки на Spotify-плейлист (`--list` / `<id>` / `--all`), чтобы плейлист можно было собрать заново |
| `make_landing_qr.py` | QR со ссылкой на лендинг → `frontend/public/landing-qr.svg`. Перегенерировать при смене домена |
| `backfill_passports.py` | Пакетная генерация паспортов оформления через Anthropic Batch API (вдвое дешевле) |
| `verify_music.py` | Чистка сохранённых подборок от несуществующих треков (`--dry-run` / `--book N`) |
| `clear_atmosphere.py` | Удаление сохранённой атмосферы, чтобы сгенерировать заново (по умолчанию — только музыка) |
| `smoke_external.py` | **Дымовой прогон перед релизом (з.83):** живой Spotify (токен+поиск+формат ответа) и Google Books. Ловит изменения контракта, которые моки не видят |
| `spotify_status.py` | Быстрая проверка «снят ли бан Spotify» (один поиск, показывает Retry-After) |
| `check_track_search.py` | Детальная диагностика поиска трека: кто найден, кто отклонён и почему |
| `backfill_atmosphere.py` | Восстановление музыки/угощений/ароматов у книг, где они пропали (инцидент 18.07) |
| `regenerate_food.py` | Массовая перегенерация еды (или `--category aroma/music`) у книг, где она уже есть — после смены промпта/модели. `--dry-run`, `--limit N`, `--book ID` |
| `reverify_music.py` | Перепроверка музыки, сохранённой при бане Spotify (з.85): резолв треков + плейлист, когда Spotify снова доступен. `--dry-run` |
| `backfill_read_dates.py` | Разовый перенос дат прочтения из CSV-выгрузки LiveLib |
| `find_covers.py` / `apply_covers.py` | Поиск недостающих обложек через Google Books и применение найденного к базе |
| `fix_openlibrary_covers.py` | Чистка битых обложек с OpenLibrary |

## Необязательные зависимости

Обложка Spotify-плейлиста (символ книги) требует
`pip install -r requirements-cover.txt` — на Linux ещё и системный Cairo
(`apt-get install libcairo2-dev`). Без них всё работает, только плейлист
получает стандартную мозаику обложек треков.

## Прежде чем запускать то, что меняет базу

```
python scripts/backup_db.py
```

Скрипты с `backfill_*` тратят токены AI или ходят во внешние API — смотрите
докстринг внутри файла, там описано поведение и ограничения.
