# Техническое задание: сервис «Библиотека книг»

**Версия:** 1.0
**Дата:** 08.07.2026
**Автор:** Ксения (pet-проект, цель — обучение работе с AI и веб-разработке)

---

## 1. Назначение и цель

Личный сервис-библиотека для «атмосферных литературных вечеров». По каждой книге сервис
подбирает сопровождение — музыку, еду/напитки и ароматы — в двух вариантах (от Claude и от
ChatGPT) с пояснениями. Дополнительно сервис ведёт учёт чтения: статусы, оценки и
рекомендации на основе прочитанного.

Проект учебный. Помимо рабочего продукта, важная цель — освоить связку «бэкенд на Python +
внешние API + AI-функции + фронтенд на React».

## 2. Пользователи

На старте — один пользователь (владелец). В модели данных заложено поле `user_id` (по
умолчанию `1`) под будущую авторизацию и многопользовательский режим.

## 3. Языки интерфейса (RU / EN) — ключевое требование

Сервис двуязычный: русский и английский. Реализуется как **единое приложение с
переключателем языка** (не два отдельных развёртывания).

3.1. **Интерфейс.** Все тексты интерфейса (подписи, кнопки, статусы, сообщения об ошибках)
вынесены в словари локализации и не «зашиты» в код. Пользователь переключает язык кнопкой
RU/EN; выбранный язык сохраняется между сессиями.

3.2. **Язык внешних запросов.** Запросы к внешним сервисам (Google Books, AI-модели) могут
формироваться как на русском, так и на английском. Язык запроса определяется одним из двух
способов (уточняется на этапе проектирования): текущим языком интерфейса либо языком самой
книги. Это влияет на:
- **Google Books:** параметры запроса и `langRestrict`; описание книги может вернуться на
  RU или EN. Нужно уметь принять описание на любом из языков и, при необходимости, показать
  перевод.
- **AI-подборки:** промпт к модели и требуемый язык ответа задаются явно, чтобы пояснения к
  подборке приходили на нужном языке.

3.3. **Хранение мультиязычного контента.** Способ хранения (одно поле с переводом «на лету»
против отдельного хранения RU- и EN-версий) фиксируется на этапе проектирования
двуязычности. Базовое правило: пользовательские данные (название, автор, оценка, статус)
языконезависимы; порождаемый контент (описание, AI-пояснения) привязан к языку.

## 4. Функциональные требования

### 4.1. Каталог книг
- Хранение книг в базе данных.
- Список книг (сетка) и карточка отдельной книги.

### 4.2. Добавление книги
- Ручное добавление по названию и автору.
- Автоподтягивание обложки и описания из внешнего источника (Google Books API).
- Если данные не найдены или источник недоступен — книга всё равно сохраняется (поля
  обложки/описания остаются пустыми, добавление не прерывается).

### 4.3. Статусы и оценки
- Статусы книги: «Хочу прочитать» (`want`), «Читаю» (`reading`), «Прочитана» (`read`).
- Оценка по шкале 1–10.
- **Правило:** оценка допустима только для статуса «Прочитана». При смене статуса с
  «Прочитана» на другой оценка сбрасывается (поведение фиксируется на этапе реализации).

### 4.4. Импорт из CSV
- Импорт книг с оценками из CSV-файла.
- Обработка ошибок формата: пропуск/отчёт по некорректным строкам без падения импорта.

### 4.5. AI-подборки «Атмосфера»
- Категории: **музыка**, **еда/напитки**, **ароматы**.
- По каждой категории — два варианта: от **Claude** и от **ChatGPT**, каждый с пояснением.
- Первым реализуется модуль **музыки** (переносится из соседнего проекта «Плейлист по
  книге»): два AI → структурированный ответ → объединение с дедупликацией и пояснениями.
- Ответы AI приходят в строгом структурированном виде (JSON) и сохраняются в БД.

### 4.6. Рекомендации
- Рекомендации новых книг на основе прочитанного и оценок.

### 4.7. Статистика и трекинг (последующие версии)
- Статистика чтения и книжный трекинг.

## 5. Нефункциональные требования

- **Платформа разработки:** Windows.
- **Отказоустойчивость внешних вызовов:** недоступность Google Books или AI-сервиса не
  должна приводить к ошибке основной операции (добавление/просмотр книги).
- **Тайм-ауты:** у всех внешних запросов задан разумный тайм-аут (ориентир — 5 секунд).
- **Обучаемость кода:** код с понятными комментариями, простая и прозрачная структура
  (приоритет над преждевременной оптимизацией).
- **Секреты:** ключи AI-сервисов хранятся вне кода (переменные окружения / `.env`, не
  попадающий в git).

## 6. Архитектура и стек

- **Бэкенд:** Python, FastAPI, SQLModel, SQLite.
- **Фронтенд:** React. Дизайн — тёплый и элегантный: янтарные акценты, «книжный» serif для
  названий.
- **Внешние сервисы:** Google Books API (обложки/описания); AI-модели Claude и ChatGPT
  (подборки «Атмосфера»).

### 6.1. Экраны фронтенда
- **Библиотека** — сетка книг.
- **Добавление книги** — модальное окно.
- **Карточка книги** — обложка, статус, оценка, описание и блок «Атмосфера» с вариантами
  Claude / ChatGPT.
- **Переключатель языка RU/EN** — доступен в общем интерфейсе.

## 7. Модель данных

### 7.1. Таблица `Book`

| Поле         | Тип             | Описание                                              |
|--------------|-----------------|-------------------------------------------------------|
| `id`         | int, PK         | Идентификатор                                          |
| `user_id`    | int (=1)        | Владелец; задел под авторизацию                        |
| `title`      | str             | Название                                               |
| `author`     | str             | Автор                                                  |
| `cover_url`  | str, nullable   | Ссылка на обложку (из Google Books)                    |
| `description`| str, nullable   | Описание (из Google Books)                             |
| `status`     | str             | `want` / `reading` / `read`                            |
| `rating`     | int, nullable   | Оценка 1–10, только для `read`                         |
| `created_at` | datetime        | Дата добавления                                        |

*На этапе двуязычности рассматривается добавление поля языка книги/описания (например,
`language`) — фиксируется при проектировании этапа RU/EN.*

### 7.2. Таблица `AISelections`

| Поле          | Тип           | Описание                                             |
|---------------|---------------|------------------------------------------------------|
| `id`          | int, PK       | Идентификатор                                        |
| `book_id`     | int, FK       | Ссылка на книгу                                       |
| `category`    | str           | `music` / `food` / `aroma`                            |
| `source`      | str           | `Claude` / `ChatGPT`                                  |
| `payload`     | JSON          | Структурированный ответ AI (например, список треков)  |
| `explanation` | str           | Пояснение к подборке                                  |
| `created_at`  | datetime      | Дата создания                                         |

*На этапе двуязычности рассматривается добавление поля языка подборки (например, `lang`),
чтобы хранить RU- и EN-варианты пояснений.*

## 8. Внешние интеграции

8.1. **Google Books API.** Поиск по названию и автору, извлечение обложки (`imageLinks`) и
описания (`description`). Приведение ссылок обложки к `https`. Учёт языка запроса и языка
ответа (см. п. 3.2).

8.2. **AI-сервисы (Claude, ChatGPT).** Формирование подборок по категориям. Требование —
структурированный ответ (JSON) для надёжного разбора и сохранения в `AISelections`. Язык
ответа задаётся явно в зависимости от выбранного языка.

## 9. Границы MVP

В MVP входит: база книг; добавление книги с автоподтягиванием обложки и описания; список и
карточка книги; статусы и оценки; одна AI-подборка — **музыка**. Двуязычность
закладывается инфраструктурно уже в MVP (минимум — переключатель RU/EN и учёт языка внешних
запросов), полное покрытие переводами — по мере развития.

Вне MVP (последующие версии): импорт CSV; AI-подборки еды и ароматов; рекомендации;
статистика и трекинг; авторизация и многопользовательский режим.

## 10. Что уже сделано

- Инициализирован репозиторий (git), настроено виртуальное окружение.
- `main.py` с моделью `Book`, база SQLite (`library.db`).
- Эндпоинты `POST /books` (с обогащением обложкой и описанием из Google Books) и
  `GET /books`. Проверено через `/docs`.

## 11. Критерии приёмки MVP

- Книгу можно добавить по названию и автору; обложка и описание подтягиваются автоматически
  (если найдены).
- Список и карточка книги корректно отображают данные.
- Статус меняется; оценка выставляется только для «Прочитана».
- Для выбранной книги можно получить музыкальную AI-подборку и увидеть её в карточке.
- Интерфейс переключается между RU и EN; внешние запросы учитывают язык.
